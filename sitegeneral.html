Box Model

*image reserved*

    "Father element" - is the tag wich the current spoken tag is inside of.

  • Border - The literal border of the text. Can be used in two manners:
    1. General Way:
      Using the general words for an superficial action, like:
      • border-top;
      • border-left;
      • border-right;
      • border-bottom;
      2. Or...
    2. Specifical Way:
      Using detailed key-words to change whatever detail or specifical point you might need in the project. Example:
      • border-top-style;
      • border-left-width;
      • border-right-color;
      • etc...
    • Just in addition:
    - Border and Outline:
    Both can do the exact same things.
    The difference? Outline is... well, Outline.

  • Box-Sizing - A concept for Box Model stetic.
    - Box-Sizing consists in the fact that the whole Box Model tags are constantly summing pixels in the box's size.
    Then, to make it all fit in a statical value of size, we use the 'Box-Sizing' propriety.
    The pattern value is the 'content-box', that define the variable size explained before. To adjust and to make this value constant, we define the box-sizing value as 'border-box'.

Block x Inline

  • Block (Block level's elements):
    1. Occupies the whole width available;
    2. Line breaker;
  • Inline (Inline level's elements):
    1. Occupies ONLY the space necessary to use the content
    2. Isn't a line breaker;
    OBS:
    • Does NOT apply width and height like the Box Model.
    • Applies margin, but only HORIZONTALLY, not vertically.
    • Padding is applicable, but generates visual bugs, like overtaking spaces inside the block.
    General Rule: NEVER use Box Model's elements in inline elements.
